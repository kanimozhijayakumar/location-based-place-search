# Location-Based Place Search using Latitude & Longitude

## Objective
To fetch nearby place information using latitude and longitude
and compare serial vs parallel API execution.

## Locations Used
- Chennai (13.0827, 80.2707)
- Bangalore (12.9716, 77.5946)
- Hyderabad (17.3850, 78.4867)

## Execution Types
1. Serial API Calls
2. Parallel API Calls

## Real-World Use Case
This approach is used in map services, food delivery apps,
and ride-hailing platforms for nearby place discovery.

## Conclusion
Parallel execution improves response time and scalability
when handling multiple location-based requests.
